# Meetup Planner

Telegram Mini App + бот для координации встреч в постоянной компании из 6 человек.

Проект решает вечную проблему «когда собираемся?»: вместо бесконечных опросов в чате каждый отмечает свою занятость в общем календаре, бот помогает выбрать время, напоминает о встречах и попутно развлекает чат. Всё управление — через Mini App прямо в Telegram, без отдельных сайтов и приложений. Проверка временем показала что активнось в чате выросала в 4 раза по сравнению со старым 'умирающим' активом.

## Что умеет

### Календарь занятости
- 14-дневная лента, где у каждого участника своя строка с цветными «пилюлями» интервалов.
- Статусы: **свободен / занят / не уверен** + уверенность 1–5 + заметка.
- Редактирование через bottom-sheet прямо в ленте.
- Бейджи на аватарках (титулы недели/дня), индикаторы дней рождения.

### Встречи
- Создание встреч с местом, временем и тегом (например, «игровая»).
- RSVP-статусы по каждому участнику, отметка «реально пришёл».
- `/next` и `/meetings` в чате — ближайшая встреча и список на неделю.
- Автоподбор времени по результатам опросов.
- Экспорт в iCalendar (подписка календарём телефона).
- Опциональный пост-фидбек: после встречи бот кидает опрос-оценку (5★) и ведёт учёт неявок.

### Опросы
- Нативные Telegram-опросы «когда встречаемся» с пресетами таймслотов.
- Кворум-автозакрытие: опрос закрывается сам, когда проголосовали все.
- Номинации игр (`/nominate`) и опросы «во что играем».

### Напоминания и авто-публикации
- Напоминания о встречах (tick-based планировщик, интервал настраивается).
- Дни рождения: уведомления за месяц / неделю / день / в день, шаблоны поздравлений с плейсхолдерами, защита от дублей.
- Еженедельные и ежедневные «титулы» участников с настраиваемыми весами, расписанием и историей.
- «Рандомные фразы»: бот собирает цитаты из истории чата и периодически постит сгенерированные из них фразы (режимы words/phrases/mix, карантин свежих сообщений, окно активности, 4 режима расписания).

### Реакции бота
- Ответы на @упоминания и реплаи.
- Реакции на мемы и альбомы: эмодзи / фраза / и то и другое, режим «подождать, дать людям отреагировать, потом с нарастающей вероятностью включиться».

### Управление ботом самим чатом
- `/zaebal` — голос за паузу бота; при пороге голосов за час бот замолкает на 3 дня.
- `/zaebal_vote` — опрос на паузу подольше (7 дней).
- Пауза снимает все авто-публикации со снапшотом настроек и сама восстанавливается по таймеру.

### Админка (внутри Mini App)
Все настройки меняются в рантайме, без редеплоя — хранятся в БД (`admin_config`):
- мастер-тумблеры всех авто-публикаций;
- генератор фраз (объёмы, окна, вероятности, расписание);
- реакции на медиа (режимы, шансы, пулы фраз и эмодзи);
- пресеты таймслотов для опросов;
- шаблоны поздравлений, веса титулов, интервалы;
- очередь задач планировщика (посмотреть/перенести/отменить), история событий;
- статус прокси и паузы.

### Надёжность доставки
Telegram API из некоторых сетей работает нестабильно, поэтому:
- **Outbox-паттерн**: авто-посты пишутся в таблицу `loser_outbox` со статусами pending/sent/failed/expired; retry-job добивает доставку до ~1 часа. Результат показывается в календаре только после реально подтверждённой отправки.
- **Smart proxy**: пул прокси с health-check'ами, автоматическое переключение direct↔proxy.
- Настраиваемые таймауты отправки (`LOSER_SEND_TIMEOUT`) под троттлинг канала.

## Стек

- **Backend:** Python 3.12, FastAPI + aiogram 3 в одном процессе (общий event loop, вебхук `/tg/webhook`), SQLAlchemy 2.0 async + Alembic, APScheduler.
- **Frontend:** React 18 + TypeScript + Vite, Telegram WebApp SDK (`@twa-dev/sdk`), Tailwind, TanStack Query, Zustand, Framer Motion.
- **БД:** PostgreSQL (Neon).
- **Хостинг:** Hugging Face Spaces (Docker, backend) + GitHub Pages (frontend) + Neon (DB) — полностью бесплатная связка.
- **Авторизация:** `Authorization: tma <initData>` — HMAC-проверка подписи Telegram WebApp, whitelist участников через env.

## Команды бота

| Команда | Что делает |
|---|---|
| `/start` | открыть Mini App, синхронизировать аватар |
| `/help` | список команд (админ видит расширенный блок) |
| `/next` | ближайшая встреча |
| `/meetings` | следующие 5 встреч с RSVP |
| `/nominate <игра>` | номинировать игру (до 5 активных) |
| `/phrase` | сгенерировать рандомную фразу вручную |
| `/loser` | ручной ролл «титула дня» (дуэль) |
| `/tasks` | очередь задач планировщика |
| `/zaebal` / `/zaebal_vote` | голосование за паузу бота |
| `/whoami` | свой Telegram ID |

## Пример: подбор настроек генератора фраз

Мини-методичка, как затюнить самую «характерную» фичу под свой чат. Все настройки — в админке Mini App, раздел **Рандомные фразы**.

**Шаг 1. Включить аккуратно.** `random_phrases.enabled = true`, режим расписания `daily_n` с параметром 1 (одна фраза в день). Окно активности по умолчанию 07:30–22:00 — ночью бот молчит.

**Шаг 2. Подобрать «связность».** Режим извлечения:
- `phrases` — целые куски сообщений, выходит осмысленнее, но ближе к прямым цитатам;
- `words` — отдельные слова, выходит абсурднее;
- `mix` (дефолт) — золотая середина.

`count_min`/`count_max` (дефолт 2–6, в среднем 4 фрагмента) — чем больше, тем длиннее и бредовее фраза.

**Шаг 3. Не дать боту попугайничать.** `lookback_days` (дефолт 7) — окно истории. `recency_quarantine_hours = 18` и `recency_quarantine_weight = 0.05` означают: сообщения свежее 18 часов почти не попадают в генерацию — бот не повторяет то, что обсуждали час назад. Если чат малоактивный, увеличьте lookback до 30 и ослабьте карантин.

**Шаг 4. Добавить непредсказуемости.** `user_chance` (дефолт 1.0) — вероятность, что джоба вообще сработает: 0.7 = бот иногда «пропускает ход», и фразы перестают быть рутиной. `collective_chance = 0.1` — шанс, что бот соберёт фразу не от одного «спикера», а из реплик случайных участников.

**Шаг 5. Расписание под ритм чата.** Когда чат привык — переключите `schedule_mode` на `random_interval` (фразы в случайные моменты внутри окна) или `fixed_times` (например, утренняя и вечерняя). Если бот начал утомлять — у чата всегда есть `/zaebal`.

Та же логика тюнинга (мастер-тумблер → консервативный дефолт → вероятности → расписание) применима к реакциям на медиа и остальным авто-публикациям.
